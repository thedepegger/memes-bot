# Troubleshooting

The most common things that go wrong, and how to fix them.

---

## Setup failures

### `python3 --version` shows 3.9 or older
Install Python 3.11+:
- **macOS**: `brew install python@3.11`
- **Ubuntu/Debian**: `sudo apt install python3.11 python3.11-venv`

Then re-run `./scripts/setup.sh`. The script auto-detects 3.11, 3.12, or 3.13.

### `./scripts/setup.sh: Permission denied`
```bash
chmod +x scripts/setup.sh
./scripts/setup.sh
```

### macOS: "Operation not permitted" on files under `~/Desktop/`
macOS Transparency-Consent-Control (TCC) blocks terminal apps from `~/Desktop`, `~/Documents`, and `~/Downloads` by default. **Two fixes:**

**Easy fix** — move the project off Desktop:
```bash
mv ~/Desktop/crypto-meme-bot ~/Code/crypto-meme-bot
cd ~/Code/crypto-meme-bot
rm -rf .venv && make setup
```

**Or** — grant your terminal access:
System Settings → Privacy & Security → Files and Folders → expand your terminal app (Terminal.app, iTerm, etc) → check **Desktop Folder**. Then quit and reopen the terminal.

### `pip install` fails with SSL errors
Your Python install has stale CA certs. Try:
```bash
/Applications/Python\ 3.11/Install\ Certificates.command
```
Or just install fresh: `brew reinstall python@3.11`.

### `pip install` fails on `sgmllib3k` or `feedparser`
Build tools missing. macOS: `xcode-select --install`. Ubuntu: `sudo apt install build-essential`.

## Runtime failures

### `pydantic_core._pydantic_core.ValidationError: ... TELEGRAM_ADMIN_USER_ID`
Your `.env` has an empty value where an integer is expected. Make sure every uncommented line has a real value or remove the line entirely.

### Telegram: "Unauthorized" / "Bad Request" / 401
- Check `TELEGRAM_BOT_TOKEN` is the full string (`12345:ABCDEF...`) — including the colon.
- Token is generated by `@BotFather`. Re-issue with `/token` if unsure.

### Telegram: bot drops candidates but my taps do nothing
- `TELEGRAM_ADMIN_USER_ID` is wrong. The bot **silently rejects** anyone who isn't the admin (no error message — by design).
- Message `@userinfobot` to confirm your numeric ID. Paste it into `.env`, restart with `make run`.

### Telegram: "chat not found" when bot tries to drop a meme
- `TELEGRAM_CHANNEL_ID` is wrong, or the bot isn't an admin of the channel.
- Channel IDs are **negative** numbers (e.g. `-1001234567890`). If yours is positive, you grabbed a user ID by mistake.
- Re-add the bot as channel admin with **Post Messages** permission.

### `TwitterRateLimitError` keeps firing
- X free tier has hidden per-15-min limits. The bot auto-reschedules approved candidates by +30min on each rate-limit hit.
- If it happens repeatedly, you're posting too fast. Reduce `POSTS_PER_DAY` in `.env`.
- If your X account got flagged: pause the bot, post manually for a week, then resume.

### X posting fails with 403 "Your client app is not configured with the appropriate oauth1 app permissions"
You forgot to enable **Read and Write** in your X app's User authentication settings. Go to developer.x.com → your app → User authentication settings → set to "Read and Write" → save → **regenerate** Access Token + Secret (the old ones are read-only) → update `.env`.

### Cryptopanic returns 401 / nothing
- Check `CRYPTOPANIC_API_KEY` in `.env`.
- The bot automatically falls back to RSS (Cointelegraph + Coindesk) if Cryptopanic is unreachable, so it'll keep working either way.

### Imgflip: `success: false` on every render
Most common cause: you're still on the **free** Imgflip tier. Upgrade to Premium ($9.99/mo) at imgflip.com → Premium. Free tier blocks programmatic renders without watermarks.

Other causes: wrong `IMGFLIP_USERNAME` / `IMGFLIP_PASSWORD` in `.env`.

### OpenRouter: `401 Authentication failed`
- Wrong or missing `OPENROUTER_API_KEY` in `.env`.
- Or: zero credit. Add $5 at openrouter.ai → Credits.

### Operator forgot to approve → backlog growing
Candidates auto-expire after 24 hours (marked `rejected` with reason `expired`). Open Telegram and clear the backlog, or use the bot's `/clear_pending` command (panic button — rejects every pending candidate).

## Test failures

### `pytest`: `ModuleNotFoundError: No module named 'src'`
Run from the project root, not from inside `tests/`:
```bash
cd /path/to/crypto-meme-bot
.venv/bin/python -m pytest
```

### Tests fail with `sqlite3.OperationalError: attempt to write a readonly database`
A previous run left a stale DB file. Wipe it:
```bash
rm -f data/memebot.db
make test
```

### Test fails with `assert ... == 'pick_template'` or similar
The test expected a function name or signature that doesn't match the implementation. This usually means a module was edited after tests were written. Read the test failure, then compare against the function's actual signature.

## Scheduler / posting issues

### Bot starts but no candidates appear after 4+ hours
- Are there news items? `sqlite3 data/memebot.db "SELECT COUNT(*) FROM news_seen;"` — if 0, the news fetcher isn't working. Check `logs/bot.log` for `job_fetch_news` errors.
- Is the LLM working? `make dry-run` — if it crashes with auth errors, fix `OPENROUTER_API_KEY` in `.env`.
- Is Imgflip working? Check `logs/bot.log` for `success: false` from Imgflip.

### Bot generates candidates but never posts to X
- `WARMUP_MODE=true` in `.env`? That's by design — flip to `false` after warmup period to enable auto-posting.
- Check `logs/bot.log` for `TwitterRateLimitError` or other tweepy errors.

### Same meme posted twice
Should be impossible — dedup is on `(template_id, topic_hash)` with a 48hr window. If it happened, check `template_usage` table:
```bash
sqlite3 data/memebot.db "SELECT * FROM template_usage ORDER BY used_at DESC LIMIT 10;"
```
File a bug if you see two posts of the same template + topic within 48hr.

## Debugging

### Watch the log live
```bash
tail -f logs/bot.log
```

### Inspect the database
```bash
sqlite3 data/memebot.db
sqlite> .schema
sqlite> SELECT id, status, generated_at, posted_at FROM candidates ORDER BY id DESC LIMIT 20;
sqlite> .quit
```

### Reset everything (nuclear option)
```bash
make clean
./scripts/setup.sh
# re-fill .env if needed
make run
```

2. Cross-reference against `docs/plan.md` (the original build spec) to see what each module is supposed to do.
3. Read `CLAUDE.md` and `.claude/rules/` — they document the architectural decisions and gotchas.
